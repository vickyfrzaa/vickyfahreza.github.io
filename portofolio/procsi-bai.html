PROCSI BAI Technical Leadership and Delivery Support

2025 Technical Leadership PT Sinergi Informatika Semen Indonesia

During my work at PT Sinergi Informatika Semen Indonesia, I was responsible for technical leadership activities on the PROCSI BAI project. My role was to support the development team by giving technical direction, reviewing implementation approaches, coordinating task execution, and helping ensure that the application was delivered with good quality and aligned with user requirements.

As a technical leader, I worked between business requirements and engineering execution. I helped translate user needs into technical plans, guided developers during implementation, reviewed code and application flow, and supported troubleshooting when the team faced technical blockers.

Project Overview

The project required continuous coordination between users, project stakeholders, and the development team. The application needed to support business workflows, user access through Microsoft Active Directory, and enterprise data integration with SAP S/4HANA while keeping the codebase maintainable and the implementation consistent with agreed development standards.

My responsibility was not only to develop features, but also to help the team make better technical decisions. This included reviewing requirement impact, planning implementation steps, checking dependencies between modules, and ensuring that integration points with Microsoft Active Directory and SAP S/4HANA were handled carefully to support daily operational use.

Key Responsibilities

  • Provided technical direction for developers during feature development and application enhancement.
  • Reviewed user requirements and translated them into technical tasks, implementation flow, and development priorities.
  • Coordinated task breakdown, development progress, and technical dependencies across application modules.
  • Reviewed code quality, application structure, data flow, and backend/frontend implementation consistency.
  • Supported debugging, troubleshooting, and technical issue resolution when blockers appeared during development or testing.
  • Supported technical alignment for integration with Microsoft Active Directory and SAP S/4HANA.
  • Collaborated with Project Manager, QA, developers, and users to ensure delivery met business expectations.

Technical Leadership Scope

My technical leadership scope covered requirement review, architecture consideration, implementation guidance, code review, issue analysis, and delivery support. I helped ensure that each feature was not only completed, but also maintainable, understandable, and compatible with the existing application flow.

I also supported the team in making practical technical decisions, such as how to structure backend logic, how to manage data validation, how to keep frontend behavior aligned with user workflow, how to handle integration dependencies, and how to reduce risks before features were released for testing or production use.

Integration Coverage

PROCSI BAI included integration with Microsoft Active Directory and SAP S/4HANA. Microsoft Active Directory supported user identity, authentication, and access alignment, while SAP S/4HANA supported enterprise data flow required by business processes.

From a technical leadership perspective, my role was to help ensure that these integrations were considered during requirement analysis, implementation planning, issue investigation, and delivery validation. This helped keep application changes aligned with enterprise system dependencies.

Leadership Contributions

  • Guided developers in choosing implementation approaches that matched project standards and business requirements.
  • Helped maintain code quality through review, feedback, and improvement suggestions.
  • Assisted the team in analyzing technical risks, dependencies, and possible impact before changes were implemented.
  • Reviewed integration impact related to Microsoft Active Directory user access and SAP S/4HANA data communication.
  • Supported QA and user testing by investigating defects and guiding fixes to the right technical area.
  • Encouraged clearer communication between technical implementation and business expectations.

Project Impact

This role helped improve delivery coordination and technical consistency across the project. By providing technical direction and review, the team could develop features with clearer implementation flow, reduce avoidable issues, and respond more effectively to changes or defects found during testing.

The experience strengthened my capability as a technical leader in balancing hands-on development understanding, team coordination, code quality, issue resolution, integration awareness, and business-focused delivery.

Technical leadership is about helping the team move with clarity: translating requirements into direction, keeping implementation quality consistent, managing integration impact, and solving technical problems before they become delivery risks.

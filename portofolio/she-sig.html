SIG Change Request Feature Development

2024 Fullstack Development PT Sinergi Informatika Semen Indonesia

During my work at PT Sinergi Informatika Semen Indonesia, I worked as a developer in the Change Request team. My main responsibility was to implement new features and improve existing application functionality based on requests from users within SIG, or Semen Indonesia Group.

The work involved understanding user needs, analyzing requested changes, developing backend logic with CodeIgniter, implementing frontend behavior with Vue.js, testing the feature flow, and fixing issues found during development or user review.

Project Overview

The Change Request workflow focused on supporting ongoing business applications used by SIG users. Instead of building only one static feature, the work required continuous improvements across application modules based on operational needs, user feedback, and business process changes.

Each request needed to be reviewed from both business and technical perspectives. I helped translate user requirements into application changes, including backend processing, database interaction, frontend components, validation rules, and interface adjustments.

Key Responsibilities

  • Developed new application features based on Change Request submissions from SIG users.
  • Implemented backend logic and data processing using CodeIgniter.
  • Built and adjusted frontend functionality using Vue.js.
  • Added validation, form behavior, data handling, and user flow improvements.
  • Supported testing, debugging, issue fixing, and feature refinement.
  • Coordinated implementation details based on user requirements and application behavior.

Technical Implementation

The backend side was developed using CodeIgniter to handle business logic, controller processes, model queries, validation rules, and data transactions. The implementation focused on keeping feature changes aligned with existing application structure while maintaining stable data flow.

On the frontend side, Vue.js was used to build and update interactive user interface behavior. This included handling form input, displaying dynamic data, improving user interactions, and connecting frontend actions with backend processes.

Change Request Scope

  • Requirement analysis from user-submitted Change Requests.
  • Feature development and enhancement for existing business applications.
  • CodeIgniter backend development for business rules and data flow.
  • Vue.js frontend development for interactive application features.
  • Testing support, debugging, bug fixing, and feature adjustment.

Project Impact

This work helped support SIG users by delivering application improvements that matched their operational needs. The Change Request process allowed business users to receive new features, workflow adjustments, and application fixes without disrupting the existing system.

The experience strengthened my ability to work with real user requirements, maintain existing applications, implement fullstack changes, and deliver practical features using CodeIgniter and Vue.js in a business environment.

Change Request development is about translating user needs into stable application improvements: clear backend logic, useful frontend behavior, proper validation, and reliable fixes for real business workflows.

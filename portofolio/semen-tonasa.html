Fullstack Web Development

Guest Accommodation & Transportation Request System

2024 Fullstack Web Development Semen Tonasa Jakarta

Fullstack Web Development is a fullstack web application developed for Semen Tonasa Jakarta to manage guest accommodation and transportation requests through a structured digital workflow. The system supports hotel room requests, vehicle requests, approval processes, and operational tracking from the initial user request until the guest service is completed.

The application was designed to replace manual request handling with a more organized web-based process. Users can submit requests through online forms, select the required accommodation or transportation details, and monitor the request status based on the approval workflow.

Project Overview

The main objective of this project was to provide an internal business application that helps manage incoming guests from Semen Tonasa or external company visitors who come to Jakarta. The system allows users to request hotel accommodation based on room levels and request operational vehicles based on pickup and drop-off requirements.

The application includes an approval workflow to ensure every request is reviewed and approved by the appropriate role before being processed further. This helps improve request control, operational visibility, and service coordination between users, approvers, and operational teams.

Key Highlights

  • Hotel accommodation request form for company guests and external visitors.
  • Room level selection to support different guest accommodation categories.
  • Vehicle request form with pickup point, destination, schedule, and trip details.
  • Multi-step approval workflow from user request, approval process, service execution, until guest checkout or transportation completion.
  • Responsive web interface built for desktop, tablet, and mobile access.
  • MySQL database structure designed to store request data, approval status, guest information, room category, vehicle details, and request history.

Workflow Coverage

The system covers two main operational workflows: hotel accommodation request and vehicle request. Each workflow starts from a user-submitted form and moves through approval, operational processing, service execution, and final completion.

Accommodation Request Flow

  • User submits a hotel or accommodation request through the web form.
  • User fills in guest information, company origin, visit purpose, stay period, and required room level.
  • Request goes through approval based on the configured workflow.
  • Approved requests are processed for room arrangement and guest accommodation.
  • Workflow is completed when the guest checks out from the accommodation.

Transportation Request Flow

  • User submits a vehicle request through the web form.
  • User fills in passenger details, pickup point, destination, schedule, and trip purpose.
  • Request is reviewed and approved based on the operational approval workflow.
  • Approved requests are processed for vehicle arrangement and pickup execution.
  • Workflow is completed after the guest has been transported back or the trip is fully completed.

Technical Implementation

The application was developed using HTML, CSS, JavaScript, jQuery, Ajax, Bootstrap, REST API, and MySQL. The frontend was built with a responsive design approach, while Ajax and REST API communication were used to support dynamic data transactions, request submission, approval status updates, and workflow processing.

The system was deployed using Hostinger and structured to support daily operational usage. The implementation focused on maintainable code, clear data flow, organized request tracking, and a user-friendly interface for both requesters and approvers.

Project Scope

My scope in this project included fullstack development, responsive interface implementation, request form development, approval workflow handling, database design, Ajax-based interaction, REST API communication, MySQL data management, testing, deployment preparation, and production support.

This project demonstrates how a fullstack web application can simplify manual operational requests by turning accommodation and transportation workflows into a structured, traceable, and approval-based digital process.
